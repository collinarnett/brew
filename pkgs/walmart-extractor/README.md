# walmart-extractor

Developer tool that discovers Walmart's current GraphQL persisted query hashes and generates the `Walmart.Internal.Endpoints` module for the `walmart` library.

```
walmart-extractor > pkgs/walmart/src/Walmart/Internal/Endpoints.hs
```

## When to run

The `walmart` library will return `WalmartHashRotated` (HTTP 400) when Walmart deploys new persisted query hashes. Run `walmart-extractor`, pipe the output to the Endpoints module, and rebuild `walmart`.

## How it works

The tool renders Walmart's orders page using lightpanda (a headless browser), extracts JavaScript bundle URLs from the HTML, downloads each bundle, and scans for `name:"OperationName",hash:"<64-char-hex>"` patterns. It matches discovered operations against the known set (`PurchaseHistoryV2`) and outputs a Haskell module with the full endpoint URLs as constants.

The `getOrder` hash is a server-side persisted query not present in the frontend JS, so it uses a seed hash that has been stable across deployments.

## Output format

```haskell
-- GENERATED by walmart-extractor. Do not edit manually.
{-# LANGUAGE OverloadedStrings #-}

module Walmart.Internal.Endpoints where

import Data.Text (Text)

purchaseHistoryUrl :: Text
purchaseHistoryUrl = "https://www.walmart.com/orchestra/cph/graphql/PurchaseHistoryV2/<hash>"

getOrderUrl :: Text
getOrderUrl = "https://www.walmart.com/orchestra/orders/graphql/getOrder/<hash>"
```
